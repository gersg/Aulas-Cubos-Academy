Fundamentos de Layout

Introdução

O que é Viewport?

A viewport refere-se à área visível de uma página web no dispositivo do usuário.

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Layout

Layout Fluido e Responsivo

Adapta-se ao tamanho da viewport, utilizando propriedades como max-width e media queries.

body {
    margin: 0;
    padding: 0;
    font-family: Arial, sans-serif;
}

.container {
    max-width: 1200px;
    margin: auto;
    padding: 10px;
    background-color: lightblue;
}

@media (max-width: 768px) {
    .container {
        padding: 5px;
    }
}

Container

Elemento usado para encapsular outros conteúdos e organizar o layout.

Exemplo: 
<div class="container">
    <h1>Título Principal</h1>
    <p>Texto de exemplo no container.</p>
</div>

Position

Define como o elemento será posicionado no layout:

  • Static: Posição padrão.
  • Relative: Relativo à sua posição normal.
  • Absolute: Relativo ao ancestral posicionado.
  • Fixed: Fixo na viewport.
  • Sticky: Combina comportamento de relative e fixed.
Exemplo de Position Relative: 
div {
    position: relative;
    top: 10px;
    left: 20px;
}

z-index

Controla a ordem de empilhamento dos elementos.

Exemplo: 
div {
    position: relative;
    z-index: 10;
}

Diferença das Tags <img>

O elemento <img> serve exclusivamente para exibir imagens, enquanto o uso de background-image em uma <div> permite adicionar texto e outros elementos.

Exemplo: 
<img src="image.jpg" alt="Imagem com <img>">

<div style="background-image: url('image.jpg'); width: 300px; height: 200px;">
    <p>Texto sobre a imagem</p>
</div>

Uso de !important

Força a aplicação de um estilo, ignorando a hierarquia do CSS.

Exemplo: 
div {
    background-color: red !important;
}
